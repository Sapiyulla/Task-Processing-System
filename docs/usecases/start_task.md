# Start Task Use Case

## Условия
 - Пользователь должен быть зарегистрирован в системе
 - Все доменные правила соблюдены:
    1. Менее 3 задач выполняется на текущий момент
    2. Менее 10 задач выполнено за текущий день

## Порядок действий (от лица пользователя)
1. Нажать на задачу со статусом `CREATED` (откроется меню выбора действий)
2. Нажать на "Запусить"

## Бизнес-логика (технически)
1. Обработка события клика на кнопку "Запустить"
2. Отправка на сервер `json`:
```json
{
    "task_id": uuid,
}
```
3. Проверка доменных правил
4. Установка задаче статуса `QUEUED` и отправка в очередь на выполнение (Kafka)
5. Рабочий процесс (**Worker**) забирает задачу из очереди и начинает выполнять, установив статус задачи - `IN_PROGRESS`
6. После завершения статус задачи устанавливается как `COMPLETED`
