# Create Task Use Case

## Условия
 - Пользователь должен быть зарегистрирован в системе
 - Все доменные правила соблюдены:
    1. Менее 70 задач в общем

## Порядок действий (от лица пользователя)
1. Клик на кнопку "Новая задача"
2. Заполнить поля: тип задачи, и входные данные [`payload`] (исходя от типа)
3. Клик `Enter` или "Создать"

## Бизнес-логика (технически)
1. Обработка события клика на кнопку "Новая задача"
2. Отправка на сервер `json` (_подробнее см. [task.md](/docs/domain/task.md)_):
```json
{
    "type": "report_generate",
    "payload": {...}
}
```
3. Проверка доменных правил
4. Создание задачи (с установкой `uuid` к задаче и статуса `CREATED`) и привязка к клиенту через `user_id`
5. Сохранение в базе данных
