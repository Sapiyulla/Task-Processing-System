# User

> Пользователь - `aggregate` клиента, который совершает определенные действия в системе.

## Аргументы
 - `id` - уникальный идентификатор
 - `name` - имя пользователя
 - `email` - почта пользователя
 - `tasks_count` - общее числовое количество задач
 - `in_progress_count` - числовое количество задач в прогрессе

## Команды
command | description
-|-
`CreateTask` | Создание задачи

## Инварианты

### Общие
- У пользователя дневной лимит задач: 10
- Одновременно не более 3 задач на 1 пользователя

### При создании
 - Не более 70 задач на 1 пользователя
