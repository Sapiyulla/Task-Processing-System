# User

> Пользователь - `aggregate` клиента, который совершает определенные действия в системе.

## Аргументы
 - `id` - уникальный идентификатор
 - `name` - имя пользователя
 - `email` - почта пользователя
 - `all_tasks` - все задачи
    > Имеет вид: `[task_id]: [status]`

## Инварианты

### Общие
- У пользователя дневной лимит задач: 10
- Одновременно не более 3 задач на 1 пользователя
