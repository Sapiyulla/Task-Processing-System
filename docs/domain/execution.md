# Execution

## Описание
Выполнение - это `aggregate`. Отвечает за выполнение **[задачи](./task.md)**, но не зависит от неё. Способен изменять _состояние_ задачи исходя из бизнес-правил.

## Атрибуты
 - `id` - уникальный идентификатор (`UUID`)
 - `task_id` - идентификатор задачи к которой привязан (`UUID`)
 - `date` - дата выполнения (`RFC3339`)
 - `status` - статус выполнения
    > Не относится к статусу **задачи**  
    > _Подробнее см. [статусы выполнения](#статусы-выполнения)_
 - `error` - ошибка, если выполнение провалилось
    > _Подробнее см. [ошибки выполнения](#ошибки-выполнения)_

### Статусы выполнения
status | description
-|-
`RUNNING` | Выполнение в процессе
`SUCCESS` | Выполнение успешно
`FAILED` | Выполнение завершилось с проблемой

### Ошибки выполнения

#### Типы
error type | description
-|-
`RetryableError` | Ошибка подлежащая повторной попытке обработки.
`NotRetryableError` | Ошибка не подлежащая повторной попытке.

## Инварианты

### Общие
 - Выполнение не более 10 минут
 - Задача завершенная с ошибкой `NotRetryableError` не может быть запущена повторно

## Команды

command | description
-|-
`StartExecution` | Начало работы над задачей
`CompleteExecution` | Завершение работы 
`FailExecution` | Завершение с ошибкой
